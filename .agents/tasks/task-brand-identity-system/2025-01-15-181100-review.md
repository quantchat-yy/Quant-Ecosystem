# Brand identity system for the Quant ecosystem (v3)

All four issues from the v2 review are resolved. QuantDrive and QuantAI logo circles lightened to `#818CF8` and `#A78BFA` respectively, yielding 6.33:1 and 6.94:1 against the dark initial text. The `light.accentForeground` and `colorblindSafe.accentForeground` switched to `#000000`, producing 6.59:1 and 7.32:1. The `dark.destructiveForeground` switched to `#000000`, producing 5.58:1. New tests validate accent/accentForeground and destructive/destructiveForeground AA compliance across all six themes. The package ships 33 passing tests, zero typecheck errors, and clean lint.

**Watch for:** No blocking concerns remain. The `generateAppIconSet` icon-set text still uses white-on-color in the small favicon/PWA icons (distinct from the logo SVGs that were fixed), but these are 10px glyphs at sub-32px sizes where WCAG text contrast doesn't meaningfully apply (possible, low-severity). The `generateFaviconSvg` and `relativeLuminance` exports remain untested, but are minor coverage gaps not blocking ship.

**Verdict**: APPROVED

## High-level view

The theme system now passes AA for all four semantic color pairs across all six themes. The fix strategy — switching foreground colors to `#000000` where the colored background is too light for white text — is consistent: light and colorblindSafe accents use dark foreground, dark-theme destructive uses dark foreground, while themes with sufficiently dark semantic colors retain white.

The logo initial fix lightens the circle fill so the existing dark text achieves AA. `#818CF8` for QuantDrive and `#A78BFA` for QuantAI stay within the same hue family while pushing luminance high enough for `#1A0F00` to clear 4.5:1. The logo circle colors now diverge from the app registry brand colors for these two apps — the registry defines the canonical hue for CSS/theming, while `appLogos` uses contrast-optimized variants for rendering.

Two new parameterized tests iterate all themes for accent and destructive contrast, bringing total coverage to all four semantic foreground/background slots.

## Logo circle color divergence from app registry

QuantDrive's brand color in the app registry remains `#6366F1`, but the logo SVG uses `#818CF8`. QuantAI retains `#8B5CF6` in the registry but uses `#A78BFA` in the logo. The `appLogos` module is the authority for logo rendering; the `apps` registry is the authority for CSS theming and icon backgrounds. Consumers who need a single "brand color" per app should use `apps[id].color`, not extract from the logo SVG.

## Remaining API asymmetry (acknowledged, not blocking)

`generateAppCSS('nonexistent')` returns a CSS comment while `generateThemeCSS('unknown')` and `generateAppIconSet('unknown')` throw. The test explicitly asserts the comment-return behavior, so this is a deliberate backward-compatibility decision. A follow-up to align all three functions can happen outside this PR.
