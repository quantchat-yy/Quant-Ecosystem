# Cross-App Orchestrator, Premium Shell Components, and Professional Auth/UX Flows

This change adds three feature clusters to the Quant ecosystem: a cross-app agentic orchestrator backed by a natural-language intent router with SSE streaming to the frontend (FEAT-001), a set of premium shell micro-features including universal search, notifications, quick actions, undo, offline indicator, and keyboard/drag-drop hooks (FEAT-002), and professional UX flows covering login, OAuth consent, onboarding wizard, settings panel, activity feed, and cross-app relations (FEAT-003). The approach wires the orchestrator into the QuantAI Next.js app via a new `/api/ask` route that streams tool execution events over SSE, consumed by an `AskQuantPanel` component. The shared-ui package gains ~10 new components and 2 hooks, all exported from barrel files. Test coverage is broad (9 new test files, ~1400 lines) but limited to server-side render assertions and orchestrator unit tests; interactive behavior and the SSE endpoint are untested.

**Watch for:** The `/api/ask` route accepts unauthenticated requests with user input forwarded directly to the orchestrator (confirmed). The SSE stream has no backpressure, timeout, or connection-limit mechanism (confirmed). The `OfflineIndicator` uses a hardcoded 2-second fake sync delay instead of observing real reconnection (confirmed).

## High-level view

The orchestrator layer (CrossAppOrchestrator + ContextManager) composes existing planner and executor machinery into a streaming facade. The design is plan, iterate steps, emit events, break on failure. The gap is that it runs fully in-process with no rate limiting or user isolation, and the API route that exposes it performs zero authentication or authorization checks.

The SSE streaming integration between `/api/ask` and `AskQuantPanel` has no reconnection logic, no keep-alive ping, and no timeout on the server stream, so a stalled orchestrator execution will hold the connection open indefinitely.

The premium shell components are purely presentational (callbacks supplied via props), which means no state persistence exists yet. The `OfflineIndicator` fabricates its sync-complete timing rather than observing real reconciliation.

The professional UX flows (LoginPage, ConsentScreen, FullOnboardingWizard, SettingsPanel, ActivityFeed, CrossAppRelations) provide the authentication and settings surface. The login page delegates actual auth to parent callbacks; no CSRF protection, rate limiting, or credential-handling logic exists in this diff.

The `useKeyboardShortcuts` hook creates a new `KeyboardShortcutManager` instance per component mount and attaches a global keydown listener. Multiple consumers each get independent manager instances, which means shortcut conflicts between components are not mediated.

## Unauthenticated orchestrator endpoint

The `/api/ask` route in `apps/quantai/src/app/api/ask/route.ts` extracts the request body and immediately constructs an orchestrator:

```ts
const userId = body.userId ?? 'anonymous';
const sessionId = body.sessionId ?? `session-${Date.now()}`;
const orchestrator = new CrossAppOrchestrator(allTools, contextManager);
const results = await orchestrator.execute(body.input, { userId, sessionId, dryRun: body.dryRun });
```

There is no session verification, no auth header check, no rate limit. Any caller can send arbitrary commands attributed to any `userId` they supply. In a production deployment this endpoint would allow unauthenticated users to trigger tool execution across the ecosystem. The `dryRun` flag is also caller-controlled, meaning an attacker can disable dry-run protection by simply not sending it.

## SSE stream lifecycle gaps

The server-side readable stream has no timeout. If `orchestrator.execute` blocks (e.g., a tool handler that never resolves), the HTTP connection remains open indefinitely:

```ts
const stream = new ReadableStream({
  async start(controller) {
    // ...
    const results = await orchestrator.execute(body.input, { ... });
    // If this never resolves, controller.close() is never called
  },
});
```

There is also no periodic keep-alive ping. Intermediate proxies or load balancers with idle timeouts will silently drop the connection. The client (`AskQuantPanel`) has no reconnection logic; once the stream ends or errors, the user must manually re-submit.

The client-side SSE parser does not handle malformed JSON from the server (a `JSON.parse` throw inside the read loop will surface as a generic error rather than a parseable event).

## User input forwarding without sanitization

`body.input` is a raw string forwarded to `IntentRouter.route(input)` and `MultiStepPlanner.plan(input, ...)`. The input length is unbounded. If the intent router or planner performs regex matching, excessively long or adversarial strings could cause catastrophic backtracking or unexpected tool selection. At minimum, enforce a maximum input length (e.g., 2000 characters) at the API boundary.

## OfflineIndicator's synthetic sync state

```ts
const handleOnline = useCallback(() => {
  setStatus('syncing');
  onStatusChange?.('syncing');
  setTimeout(() => {
    setStatus('online');
    onStatusChange?.('online');
  }, 2000);
}, [onStatusChange]);
```

The 2-second timeout is unconditional. If real sync takes 10 seconds (or 0 seconds), the indicator lies. This component should accept a `syncPromise` or `isSyncing` prop from the application layer rather than fabricating the timing internally.

## UniversalSearch debounce/unmount race

The debounced search effect fires an async callback after 200ms:

```ts
debounceRef.current = setTimeout(async () => {
  const searchResults = await onSearch(query);
  setResults(searchResults); // <-- may fire after unmount
  setIsLoading(false);
}, 200);
```

The cleanup function clears the timeout but cannot cancel an already-dispatched `onSearch`. If the modal is closed (unmounted) while a search is in-flight, React will warn about state updates on an unmounted component. A stale-request guard (incrementing a generation counter, or using `AbortController`) would fix this.

## Test coverage gaps

All UI tests use `renderToStaticMarkup` (server-only), which validates structure and accessibility attributes but cannot exercise:

- Keyboard navigation (UniversalSearch arrow keys, QuickActions keyboard loop)
- Drag-and-drop events (useDragDrop)
- SSE streaming in AskQuantPanel (requires mocking fetch + ReadableStream)
- Timer-based behavior (UndoToast countdown, OfflineIndicator timeout)

The hooks (`useKeyboardShortcuts`, `useDragDrop`) have zero test files. The orchestrator tests are thorough for the happy path and failure-break behavior but do not test concurrent executions or listener memory-leak scenarios.
